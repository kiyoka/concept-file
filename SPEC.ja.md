# Concept File フォーマット仕様

**バージョン:** 0.1.0 (ドラフト)
**ステータス:** ドラフト
**拡張子:** `.concept`
**MIME Type:** `application/x-concept`

## 概要

Concept File (`.concept`) は、テキスト・埋め込みベクトル・メタデータ・出所情報(provenance)を単一のポータブルなファイルにまとめるプレーンテキストファイルフォーマットです。知識表現の原子単位として機能し、外部データベースを必要とせずにセマンティック検索やRAGワークフローを実現します。ファイル全体が人間に読める形式で、任意のテキストエディタで開くことができます。

## 設計目標

1. **自己完結** -- テキスト、埋め込みベクトル、出所情報が1ファイルに共存する。
2. **ポータブル** -- ファイルをコピーするだけで知識を移動できる。データベースの移行は不要。
3. **自己記述** -- 各ファイルがどのように生成されたか(モデル、パイプライン、ソース)を記録する。
4. **人間が読める** -- ファイル全体がプレーンテキスト。任意のエディタで開き、gitでdiffし、`cat`で確認できる。
5. **合成可能** -- ファイルシステム上のファイル群がそのままKnowledge Baseを形成する。標準ツール(`find`, `sort`, CLIパイプ)がそのまま使える。

## ファイルレイアウト

`.concept`ファイルは2つのセクションで構成されます:

```
+==============================+
|  ヘッダ行 (1行)               |  "CNCP v1 <json_length>\n"
+==============================+
|  JSON本体 (複数行)            |  pretty-printed UTF-8 JSON
+==============================+
```

### 1. ヘッダ行

ファイルの先頭行は、改行(`\n`, `0x0A`)で終端されるプレーンテキストのヘッダです。

フォーマット: `CNCP v<version> <json_length>\n`

| トークン          | 説明                                                  |
|------------------|-------------------------------------------------------|
| `CNCP`           | マジック識別子。常にリテラル文字列 `CNCP`。                |
| `v<version>`     | フォーマットバージョン。現行: `v1`。                      |
| `<json_length>`  | 10進整数。JSON本体のバイト長(ヘッダ行の`\n`以降からファイル末尾まで)。 |

例: `CNCP v1 1432\n`

この設計により、`head -1 file.concept` でフォーマット識別子とバージョンが確認でき、`file`コマンドも先頭の `CNCP` 文字列でフォーマットを判定できます。

### 2. JSON本体

UTF-8エンコードされたJSONオブジェクト。JSONはpretty-print(インデント付き複数行)でもコンパクト(単一行)でも構いません。実装は両方の形式を受け入れなければなりません(MUST)。

埋め込みベクトルを含むすべてのデータがこのJSONオブジェクト内に格納されます。

#### 必須フィールド

| フィールド    | 型      | 説明                                         |
|-------------|--------|----------------------------------------------|
| `concept`   | string | 概念の人間が読める名前                          |
| `version`   | string | このコンテンツのセマンティックバージョン (例: `"1.0"`) |
| `created_at`| string | 作成日時 (ISO 8601形式)                        |

#### コンテンツフィールド

| フィールド   | 型       | 必須 | 説明                              |
|------------|----------|------|-----------------------------------|
| `text`     | string   | Yes  | 主要なテキスト表現                   |
| `summary`  | string   | No   | 概念の短い要約                      |
| `keywords` | string[] | No   | キーワードまたはタグ                 |
| `language` | string   | No   | BCP 47言語タグ (例: `"ja"`, `"en"`) |

#### 埋め込み

| フィールド           | 型        | 必須  | 説明                                  |
|--------------------|----------|------|---------------------------------------|
| `embedding`        | object   | No   | 埋め込みベクトルとそのメタデータ          |
| `embedding.model`  | string   | Yes* | 埋め込み生成に使用したモデル              |
| `embedding.dim`    | uint     | Yes* | 次元数                                 |
| `embedding.vector` | number[] | Yes* | 浮動小数点数の配列                      |

\* `embedding`オブジェクトが存在する場合に必須。

#### 出所情報 (Provenance)

| フィールド                | 型      | 必須 | 説明                                          |
|-------------------------|--------|------|-----------------------------------------------|
| `provenance`            | object | No   | この概念がどのように生成されたかを記録            |
| `provenance.source_url` | string | No   | 元の素材のURL                                  |
| `provenance.pipeline`   | string | No   | パイプラインの記述 (例: `"fetch \| extract_text \| summarize \| embed"`) |
| `provenance.llm_model`  | string | No   | テキスト生成・要約に使用したLLM                   |

#### リレーション

| フィールド            | 型       | 必須  | 説明                                        |
|---------------------|----------|------|---------------------------------------------|
| `relations`         | array    | No   | 他のconceptファイルへのリンク                   |
| `relations[].type`  | string   | Yes* | 関係の種類: `"broader"`, `"narrower"`, `"related"`, `"instance"` |
| `relations[].ref`   | string   | Yes* | 参照先の`.concept`ファイルへの相対パス           |

\* リレーションエントリが存在する場合に必須。

#### 例

```
CNCP v1 1021
{
  "concept": "日本のAIスタートアップ動向",
  "version": "1.0",
  "created_at": "2026-03-14T10:00:00Z",

  "text": "日本のAIスタートアップは2024年以降急増しており...",
  "summary": "日本AIスタートアップの資金調達・技術動向の概要",
  "keywords": ["AI", "スタートアップ", "日本", "LLM"],
  "language": "ja",

  "embedding": {
    "model": "text-embedding-3-small",
    "dim": 1536,
    "vector": [0.0234, -0.1823, 0.0091, ...]
  },

  "provenance": {
    "source_url": "https://example.com/article",
    "pipeline": "fetch | extract_text | summarize | embed",
    "llm_model": "claude-sonnet-4-6"
  },

  "relations": [
    { "type": "related", "ref": "japan_tech_policy.concept" },
    { "type": "broader", "ref": "global_ai_trends.concept" }
  ]
}
```

## 読み取りアルゴリズム

```
1. 最初の行(\nまで)を読み取り。"CNCP v<version> <json_length>" としてパース。
2. magic == "CNCP" を検証。バージョン互換性を確認。
3. json_lengthバイトを読み取り(またはEOFまで読み取り)。UTF-8 JSONとしてデコード。
4. パース済みJSONオブジェクトからフィールドに直接アクセス。
```

標準的なテキストツールでも内容を確認できます:

```bash
head -1 file.concept                   # フォーマットバージョンを表示
cat file.concept                       # ファイル全体が人間に読める
tail -n +2 file.concept | jq .concept  # 概念名を抽出
grep '"concept"' file.concept          # ファイル群をまたいだ検索
```

## ファイルサイズの目安

| 内容                                 | おおよそのサイズ  |
|-------------------------------------|-----------------|
| ヘッダ行                              | 約15 B          |
| JSON本体 (埋め込みなし)                | 0.5 - 2 KB     |
| 埋め込み (1536次元、JSON配列)          | 約15 KB         |
| 埋め込み (3072次元、JSON配列)          | 約30 KB         |
| **典型的な合計 (埋め込みあり)**         | **約16 - 32 KB**|
| **典型的な合計 (埋め込みなし)**         | **約0.5 - 2 KB**|

## 既存フォーマットとの比較

| フォーマット    | テキスト | 埋め込み | 出所情報 | 自己記述 | 人間が読める |
|--------------|---------|---------|---------|---------|------------|
| .txt         | Yes     | No      | No      | No      | Yes        |
| .json        | Yes     | 手動    | 手動     | No      | Yes        |
| Parquet      | Yes     | Yes     | No      | No      | No         |
| **.concept** | Yes     | Yes     | Yes     | Yes     | Yes        |

## 将来の拡張 (参考情報)

以下の拡張が想定されていますが、まだ仕様化されていません:

- **複数埋め込み** -- 例: テキスト埋め込み + 画像埋め込み(CLIP)を同一ファイルに保持し、クロスモーダル検索を可能にする。
- **マルチモーダルコンテンツ** -- `text`と並列して`image_url`、`audio_url`フィールドを持つ。
- **チャンク化された概念** -- 単一のソースドキュメントを複数の`.concept`ファイルに分割し、`relations`でリンクする。
- **コレクションマニフェスト** -- `.concept`ファイルのディレクトリをインデックスする`.concept-collection`ファイルで、検索を高速化する。

## 規約

- ファイル拡張子: `.concept`
- 推奨ディレクトリ構成: `knowledge/<トピック名>.concept`
- 概念名は説明的で人間が読めるものにすること。
- `text`が変更された場合、一貫性を維持するために`embedding`を再生成すべき(SHOULD)。CIフックにより、テキストの変更に対応する埋め込みの更新がないことを検出して強制できる。
