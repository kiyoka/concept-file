
## プロジェクトについて

「ベクターデータベースなしのポータブルなセマンティック検索。ファイルとfindとsortだけ」を実現するファイルフォーマット定義と、CLIツール群を開発するプロジェクトです。

## ディレクトリ構成

```
concept-file/
├── SPEC.md
├── README.md
├── python/
│   └── concept_file/
│       ├── __init__.py
│       ├── reader.py      ← .conceptの読み書き
│       └── search.py      ← コサイン類似度検索
└── cli/
    ├── concept-embed      ← テキスト → .concept生成
    ├── concept-search     ← 自然言語でセマンティック検索
    ├── concept-sim        ← 類似度計算
    ├── concept-grep       ← ソースファイルのセマンティック grep
    ├── concept-show       ← 中身を人間が読める形で表示
    └── concept-plot       ← UMAPで可視化
```

質問です。
このツールは、コーディングエージェントから見て、skillsとして提供されていたら、どのように有効活用できそうですか？
コードの検索に使う観点と、トークン数の節約という観点から考えてみてください。
また、コーディングするにあたって、どのようなコマンドラインがあれば役に立つか考えてください。

### 回答: コーディングエージェントのskillsとしての活用

#### 1. コード検索での活用

現在、コーディングエージェントがコードを探すときは Grep（キーワード一致）と Glob（ファイル名パターン）に頼っている。これには限界がある：

- 「認証に関連するファイルはどれか」→ `grep "auth"` では `verifyPassword` や `login` を見逃す
- 「エラーハンドリングをしている箇所」→ キーワードが多様すぎて網羅できない
- リファクタリングの影響範囲分析 → 意味的に関連するファイルの特定が難しい

`concept-grep`がskillとして使えれば、意味ベースでコードを発見できる。特に：

- ユーザーの曖昧な指示（「決済まわりを直して」）から関連ファイルを一発で特定
- 未知のコードベースでの初期探索が格段に速くなる
- Grep → 0件 → 別のキーワードで再試行、というループが不要になる

#### 2. トークン数の節約

これが最も大きなメリット。現在のエージェントの探索パターン：

```
Grep "keyword1" → 大量の結果 → Read file1 → Read file2 → Read file3 → ...
```

各ステップでトークンを消費する。特に Explore Agent を使うと、サブエージェントが何十ファイルも読んで探索するため、大量のトークンを消費する。

`concept-grep`を使えば：

```
concept-grep "意味的なクエリ" -r src/ → 関連ファイル上位5件 → Read 必要なファイルだけ
```

- 無関係なファイルを読まずに済む → トークン大幅削減
- 特に大規模コードベースで効果が大きい
- `--index` は初回のみ、以降はハッシュベースのスキップで高速

#### 3. コーディングエージェント向けに欲しいコマンドライン

##### (a) `concept-grep --summary` / `--oneline`

ファイルパスだけでなく、各ファイルの構造サマリ（tree-sitterで抽出済みの `embed_source`）を一緒に返すモード。エージェントはこれを見て「このファイルを詳しく読むべきか」を判断できる。

```bash
concept-grep -r --summary "user authentication" src/
# src/AuthService.java (0.85) — class AuthService: login(), verifyToken(), ...
# src/User.java (0.72) — class User: email, verifyPassword(), ...
```

これにより Read する前に中身を概観できるのでトークン節約になる。

##### (b) `concept-grep --json`

エージェントが結果をパースしやすいJSON出力。スコア、ファイルパス、サマリをまとめて返す。

```json
[{"path": "src/Auth.java", "score": 0.85, "summary": "class AuthService..."}]
```

##### (c) `concept-grep --context` / `--snippet`

関連度の高い部分だけを抜粋して返すモード。ファイル全体をReadせずに、意味的に関連する箇所だけを得られれば、トークンを劇的に節約できる。

##### (d) `concept-diff` （新コマンド案）

```bash
concept-diff --since HEAD~3 -r src/
```

直近のgit変更に意味的に関連するファイルを見つける。「この変更の影響を受けそうなファイル」を自動的に洗い出せれば、レビューやテスト対象の特定に使える。

##### (e) `concept-grep --index --watch` （インデックスの自動更新）

エージェントのセッション中にファイルを編集するたびにインデックスが古くなる。変更ファイルだけ自動で再インデックスする仕組みがあると、セッション中ずっと正確な検索ができる。

#### まとめ

最大の価値は「探索のトークンコスト削減」。現在のエージェントは「正しいファイルを見つける」ために多くのトークンを使っているが、セマンティック検索があれば、最初から的確なファイルにたどり着ける。特に `--summary` のような、ファイルを開かずに概要を得られる機能がエージェント連携では重要。
